Mac Studio 本機開發計畫與開發文檔

For Hermes: Use subagent-driven-development skill to implement this plan task-by-task.

Goal: 在 Mac Studio 上建立一套可長期維護、可測試、可部署的本機開發工作流。

Architecture: 以 Mac Studio 作為主要開發與測試機，使用 Git 管理版本，使用 Docker/本機 runtime 隔離服務，使用 .env 管理機密設定，所有功能以小步提交、測試先行、文件同步更新為原則。

Tech Stack: macOS、Homebrew、Git、VS Code/Cursor、Docker Desktop 或 OrbStack、Node.js/Python、SQLite/PostgreSQL、pytest/vitest、GitHub。

1. 適用範圍

本文件適用於你在 Mac Studio 上進行以下類型開發：

  • Web app / API service
  • Discord / Telegram bot
  • AI agent / LLM 工具整合
  • Python / Node.js 後端服務
  • 本機原型驗證後部署到雲端 VPS 或容器平台

如果實際專案已有技術棧，以下內容可按專案調整。

2. 開發原則

  1. 本機可重現：新機器照著文檔可重新搭建環境。
  2. 小步提交：每完成一個可驗證任務就 commit。
  3. 配置與密鑰分離：.env 不進 Git；提供 .env.example。
  4. 測試先行：核心邏輯先寫測試，再實作。
  5. 文檔同步：每新增功能，同步更新 README.md 或 docs/。
  6. 避免一次性大改：每次只處理一個明確目標。

3. Mac Studio 基礎環境

Task 1: 安裝命令列工具

Objective: 準備 macOS 基礎編譯與 Git 工具。

Command:

xcode-select --install

Verify:

git --version
clang --version

Expected: 能正常輸出版本號。

Task 2: 安裝 Homebrew

Objective: 使用 Homebrew 管理開發依賴。

Command:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Apple Silicon Mac Studio 通常需要加入：

eval "$(/opt/homebrew/bin/brew shellenv)"

建議寫入 ~/.zprofile。

Verify:

brew doctor
brew --version

Task 3: 安裝常用工具

Objective: 安裝開發常用 CLI。

Command:

brew install git gh jq curl wget tree ripgrep fd direnv just

Verify:

gh --version
jq --version
rg --version

Task 4: 安裝編輯器與容器工具

Objective: 建立日常開發環境。

Command:

brew install --cask visual-studio-code
brew install --cask docker

可選替代：

brew install --cask cursor
brew install --cask orbstack

Recommendation:

  • 一般 Docker 相容需求：Docker Desktop
  • 更輕量、Mac 體驗更好：OrbStack

4. 專案目錄規範

建議統一放在：

mkdir -p ~/Developer
cd ~/Developer

單一專案結構：

my-project/
  README.md
  docs/
    development.md
    deployment.md
    architecture.md
  src/
  tests/
  scripts/
  .env.example
  .gitignore
  package.json 或 pyproject.toml
  docker-compose.yml

5. Git 與 GitHub 工作流

Task 5: 設定 Git 身分

git config --global user.name "你的名字"
git config --global user.email "你的信箱"
git config --global init.defaultBranch main

Task 6: 登入 GitHub CLI

gh auth login

選擇：

  • GitHub.com
  • HTTPS 或 SSH 均可
  • 用瀏覽器登入

Verify:

gh auth status

Commit 規範

使用 Conventional Commits：

feat: 新功能
fix: 修 bug
docs: 文件
test: 測試
refactor: 重構
chore: 工具或設定

6. Python 專案標準流程

Task 7A: 建立 Python 專案

mkdir my-python-project
cd my-python-project
python3 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip

建議依賴：

pip install pytest ruff python-dotenv

目錄：

src/my_app/
  __init__.py
  main.py
tests/
  test_main.py

測試：

pytest -q
ruff check .

7. Node.js 專案標準流程

Task 7B: 建立 Node.js 專案

安裝 Node：

brew install nvm
mkdir -p ~/.nvm

加入 shell 設定後：

nvm install --lts
nvm use --lts

建立專案：

mkdir my-node-project
cd my-node-project
npm init -y
npm install dotenv
npm install -D typescript tsx vitest eslint prettier

測試：

npm test
npm run lint

8. 環境變數與密鑰管理

建立 .env.example：

APP_ENV=development
LOG_LEVEL=debug
DATABASE_URL=sqlite:///local.db
API_KEY=replace-me

本機建立 .env：

cp .env.example .env

.gitignore 必須包含：

.env
*.local
.DS_Store
node_modules/
.venv/
__pycache__/

規則：

  • .env 不提交。
  • .env.example 必須提交。
  • 密鑰輪換後同步更新部署環境。

9. 本機服務啟動方式

建議提供 justfile：

setup:
    echo "install dependencies here"

dev:
    echo "start dev server here"

test:
    echo "run tests here"

lint:
    echo "run lint here"

之後統一使用：

just setup
just dev
just test
just lint

10. Docker / 資料庫策略

如果只需要輕量本機資料：

  • 優先 SQLite
  • 避免過早引入 PostgreSQL

如果需要 PostgreSQL，建立 docker-compose.yml：

services:
  postgres:
    image: postgres:16
    environment:
      POSTGRES_USER: app
      POSTGRES_PASSWORD: app
      POSTGRES_DB: app
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data

volumes:
  postgres_data:

啟動：

docker compose up -d

驗證：

docker compose ps

11. 測試與驗收標準

每個功能完成前必須通過：

# Python
pytest -q
ruff check .

# Node.js
npm test
npm run lint

驗收清單：

  • [ ] 功能符合需求
  • [ ] 測試覆蓋核心邏輯
  • [ ] 錯誤情境有處理
  • [ ] .env.example 已更新
  • [ ] README / docs 已更新
  • [ ] 沒有提交密鑰
  • [ ] 可以從零啟動專案

12. 開發階段計畫

Phase 0: 需求確認

輸出：

  • 功能列表
  • 使用者流程
  • 技術棧
  • 資料模型草案
  • 驗收條件

Phase 1: 專案初始化

輸出：

  • Git repo
  • 基礎目錄
  • README
  • .env.example
  • 測試框架
  • lint/format 工具

Phase 2: 最小可用版本 MVP

輸出：

  • 一條主流程可跑通
  • 本機可啟動
  • 有基本測試
  • 有錯誤日誌

Phase 3: 功能擴展

輸出：

  • 按優先級逐項實作
  • 每項功能獨立 commit
  • 每項功能有測試

Phase 4: 穩定化

輸出：

  • 補齊 edge cases
  • 改善錯誤訊息
  • 增加日誌
  • 整理文件

Phase 5: 部署準備

輸出：

  • Dockerfile 或部署腳本
  • production .env 規格
  • backup/restore 策略
  • 監控與重啟策略

13. 開發日常操作流程

每次開工：

cd ~/Developer/my-project
git pull
just dev

開新功能：

git checkout -b feat/feature-name

完成後：

just test
just lint
git status
git add .
git commit -m "feat: implement feature name"
git push -u origin feat/feature-name

14. 風險與對策

風險 對策
Mac 本機環境不可重現 用 README、justfile、Docker 固化流程
密鑰誤提交 .gitignore、.env.example、提交前檢查
功能越做越大 Phase 切分、小步 commit
測試缺失 每個核心功能先寫測試
Apple Silicon 相容問題 優先使用 arm64 image；必要時指定 platform
本機能跑、部署不能跑 提早引入 Docker 或 CI 驗證

16. 下一步

如果專案尚未建立：

  1. 確定技術棧。
  2. 在 ~/Developer 建立 repo。
  3. 建立 README、.env.example、測試框架。
  4. 實作第一條 MVP 主流程。
  5. 每完成一項功能就測試、提交、更新文件。

如果專案已存在：

  1. 先檢查現有目錄與依賴。
  2. 補 .env.example、README、測試命令。
  3. 補本機啟動文檔。
  4. 再進入功能開發。
